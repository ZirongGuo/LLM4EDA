## 模块名称
riscv_core (RISC-V Core Top Level)

## 逻辑类型
时序逻辑（顶层模块）

注意：作为顶层模块，包含 clk_i 输入端口。每个周期调用所有子模块的 process() 函数，代表一个时钟上升沿。

## 功能概述
RISC-V 处理器核顶层模块，集成 17 个子模块构成完整的 5 级流水线 RV32IM 处理器核。包含取指（Fetch）、译码（Decode）、发射（Issue）、执行（Exec/ALU/LSU/CSR/乘法/除法）和写回（Writeback）流水线阶段。支持外部内存接口、中断输入，并通过参数配置支持 M/S 模式、MMU、乘法除法扩展等功能。

## 子模块调用

### 1. u_fetch (riscv_fetch)
取指单元：负责从指令缓存获取指令，管理 PC 和分支跳转。

### 2. u_decode (riscv_decode)
译码级：缓存 Fetch 输出，内部包含 riscv_decoder 解码指令类型。

### 3. u_issue (riscv_issue)
发射单元：核心流水线控制，负责操作数获取、数据前递、指令分发和写回处理。

### 4. u_exec (riscv_exec)
执行单元：执行 ALU 运算和分支条件判断，内部包含 riscv_alu。

### 5. u_lsu (riscv_lsu)
加载存储单元：处理所有访存指令，通过 MMU 访问数据内存。

### 6. u_csr (riscv_csr)
CSR 单元：处理 CSR 指令、异常和中断，内部包含 riscv_csr_regfile。

### 7. u_mul (riscv_multiplier)
乘法器：执行 MUL/MULH/MULHSU/MULHU 指令。

### 8. u_div (riscv_divider)
除法器：执行 DIV/DIVU/REM/REMU 指令。

### 9. u_mmu (riscv_mmu)
内存管理单元：虚实地址转换和权限检查，连接指令/数据缓存接口。

## 功能说明

### 1. 流水线架构
5 级流水线：Fetch → Decode → Issue → Execute/CSR/LSU/Mul/Div → Writeback

### 2. 内存接口
- 指令内存接口：mem_i_*，通过 MMU 访问外部指令存储器
- 数据内存接口：mem_d_*，通过 MMU 访问外部数据存储器
- 支持缓存一致性操作（flush、invalidate、writeback）

### 3. 中断处理
- 通过 intr_i 接收外部中断
- CSR 模块管理中断使能和委托
- 中断响应后流水线冲刷并跳转到中断向量

### 4. 内部信号互连
- 各子模块之间通过 wire 信号连接
- 分支请求和 PC 从 Exec/CSR 通过 Issue 转发到 Fetch
- 写回数据从各执行单元汇集到 Issue 和 CSR
- MMU 连接 Fetch/LSU 和外部内存接口
