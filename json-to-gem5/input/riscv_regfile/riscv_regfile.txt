## 模块名称
riscv_regfile (RISC-V Register File)

## 逻辑类型
时序逻辑（子模块）

注意：作为子模块，其 process() 调用由顶层模块每个周期触发，代表一个时钟上升沿。不需要 clk 输入端口和时钟边沿检测逻辑。

## 功能概述
RISC-V 通用寄存器文件模块，存储 32 个 32 位通用寄存器（x0-x31）。支持两个读端口和一个写端口（2R1W）。x0 寄存器硬连线为 0，写入 x0 被忽略。模块提供寄存器读取、写入功能，并支持通过前递路径解决数据冒险。

## 端口定义

### 输入信号
| 端口名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| ra_idx_i | 4:0 | input | 读端口 A 寄存器索引 |
| rb_idx_i | 4:0 | input | 读端口 B 寄存器索引 |
| rd_wen_i | 1 | input | 写使能 |
| rd_idx_i | 4:0 | input | 写端口寄存器索引 |
| rd_data_i | 31:0 | input | 写数据 |
| bypass_ra_valid_i | 1 | input | 读端口 A 前递有效 |
| bypass_rb_valid_i | 1 | input | 读端口 B 前递有效 |
| bypass_ra_data_i | 31:0 | input | 读端口 A 前递数据 |
| bypass_rb_data_i | 31:0 | input | 读端口 B 前递数据 |

### 输出信号
| 端口名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| ra_data_o | 31:0 | output | 读端口 A 数据 |
| rb_data_o | 31:0 | output | 读端口 B 数据 |

## 功能说明

### 1. 寄存器组
- 32 个 32 位通用寄存器（x0-x31）
- x0 硬连线为 0，写入 x0 被忽略
- 支持 RV32I 架构的通用寄存器规范

### 2. 读操作
- 两路独立读端口 ra_idx_i 和 rb_idx_i
- 根据寄存器索引输出对应寄存器值
- x0 索引始终返回 0

### 3. 写操作
- 当 rd_wen_i 有效时写入 rd_idx_i 对应的寄存器
- 写入 x0 被忽略（保持为 0）
- 写操作在 process() 中同步更新

### 4. 数据前递支持
- bypass_ra_valid_i/rb_valid_i 指示前递数据有效
- 当前递有效时，前递数据优先于寄存器文件数据
- 用于解决流水线 RAW 冒险

## 子模块调用
无子模块调用。
