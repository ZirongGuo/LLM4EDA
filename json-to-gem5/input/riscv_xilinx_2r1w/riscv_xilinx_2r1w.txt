## 模块名称
riscv_xilinx_2r1w (Xilinx 2-Read 1-Write Register File)

## 逻辑类型
时序逻辑（子模块）

注意：作为子模块，其 process() 调用由顶层模块每个周期触发，代表一个时钟上升沿。不需要 clk 输入端口和时钟边沿检测逻辑。

## 功能概述
Xilinx 风格的 2 读 1 写（2R1W）寄存器文件模块，是 riscv_regfile 的备选实现。使用 Xilinx 块 RAM（BRAM）原语实现双端口读和单端口写的寄存器文件。当参数 SUPPORT_REGFILE_XILINX 使能时，使用本模块替代标准寄存器文件实现。

## 端口定义

### 输入信号
| 端口名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| clk_i | 1 | input | 时钟信号 |
| ra_idx_i | 4:0 | input | 读端口 A 寄存器索引 |
| rb_idx_i | 4:0 | input | 读端口 B 寄存器索引 |
| rd_wen_i | 1 | input | 写使能 |
| rd_idx_i | 4:0 | input | 写端口寄存器索引 |
| rd_data_i | 31:0 | input | 写数据 |

### 输出信号
| 端口名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| ra_data_o | 31:0 | output | 读端口 A 数据 |
| rb_data_o | 31:0 | output | 读端口 B 数据 |

## 功能说明

### 1. Xilinx BRAM 实现
- 使用 Xilinx 双端口 BRAM 原语
- 两个独立的读端口（异步读）
- 一个写端口（同步写）
- 32 个 32 位寄存器映射到 BRAM

### 2. 读操作
- 两路独立读端口
- 异步读取：读地址变化后立即输出数据
- 与标准寄存器文件行为一致

### 3. 写操作
- 同步写：在时钟上升沿写入
- 写使能控制
- x0 写入被忽略（硬连线为 0）

### 4. 与标准 regfile 的区别
- 使用 FPGA BRAM 资源而非触发器实现
- 更适合 FPGA 综合
- 读操作为异步，写操作为同步
- 不支持内部数据前递

## 子模块调用
无子模块调用。
