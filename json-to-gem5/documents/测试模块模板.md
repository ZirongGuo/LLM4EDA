# 1. 组合逻辑模块测试模板

组合逻辑模块的TestGenerator无需事件调度，在startup()中直接同步执行所有测试。

**目录结构**：
```
gem5/src/generators/xxx/xxx_test/
├── XXXTestGenerator.py      # Python配置
├── xxx_test_generator.hh   # C++头文件
├── xxx_test_generator.cc   # C++实现
└── SConscript              # 构建配置
```

**XXXTestGenerator.py**：
```python
from m5.params import *
from m5.SimObject import SimObject

class XXXTestGenerator(SimObject):
    type = "XXXTestGenerator"
    cxx_header = "generators/xxx/xxx_test/xxx_test_generator.hh"
    cxx_class = "gem5::XXXTestGenerator"

    dut = Param.XXX("Device under test")
```

**xxx_test_generator.hh**：
```cpp
#include "generators/xxx/xxx.hh"
#include "params/XXXTestGenerator.hh"
#include "sim/sim_object.hh"

namespace gem5 {

class XXXTestGenerator : public SimObject {
private:
    XXX& dut;
    struct TestVector { ... };
    std::vector<TestVector> testVectors;
    size_t passed;
    size_t failed;
    void runTests();

public:
    XXXTestGenerator(const XXXTestGeneratorParams &params);
    void startup() override;
};

} // namespace gem5
```

**xxx_test_generator.cc（组合逻辑特点）**：
```cpp
void
XXXTestGenerator::runTests()
{
    for (size_t i = 0; i < testVectors.size(); i++) {
        dut.setInput(vec.value);
        dut.process();  // 立即计算，无需等待
        auto result = dut.getOutput();
        // 验证结果
    }
    if (failed > 0) panic("Some tests failed!");
    exitSimLoop("XXXTestGenerator complete");
}

void
XXXTestGenerator::startup()
{
    runTests();  // 直接运行，无需事件调度
}
```

**configs/xxx_test/xxx_test.py**：
```python
import m5
from m5.objects import *

root = Root(full_system=False)
root.dut = XXX()
root.tester = XXXTestGenerator(dut=root.dut)
m5.instantiate()
exit_event = m5.simulate()
```
# 2. 时序逻辑子模块测试模板
可以参考组合逻辑测试模板，原因如下
```
2.组合逻辑模块的类结构如下
-输入输出端口作为成员变量
-输入端口的set函数
-输出端口的get函数
-process()函数，该函数无输入无返回值，仅通过输入端口的值计算输出的值，并把该值赋给输出端口

使用过程大致为：先通过set函数给输入端口赋值，调用process()函数计算，计算后通过get函数获取输出端口的值

3.子模块时序逻辑模块的类结构如下
-输入输出端口作为成员变量
-状态寄存器作为成员变量
-输入端口的set函数
-输出端口的get函数
-process()函数，该函数无输入无返回值，仅通过输入端口的值计算输出的值，并把该值赋给输出端口

结构与组合逻辑实现基本类似，唯一的区别是有内部变量存储时序状态，时钟是靠每个周期顶层模块调用process()函数
使用过程大致为：先通过set函数给输入端口赋值，调用process()函数计算，计算后通过get函数获取输出端口的值
```


# 3. 使用子模块的顶层模块测试模板

当顶层模块（如TopDecoder）使用子模块时，需要在Python配置中先实例化所有子模块，然后通过Param传递给顶层模块。

**TopDecoder.py（带子模块参数）**：
```python
from m5.params import *
from m5.SimObject import SimObject

class TopDecoder(SimObject):
    type = "TopDecoder"
    cxx_header = "generators/top_decoder/top_decoder.hh"
    cxx_class = "gem5::TopDecoder"

    ch_evidence = Param.ChEvidence("Channel evidence module")
    var_to_check = Param.VarToCheck("Variable to check module")
    check_to_var = Param.CheckToVar("Check to var module")
    belief = Param.Belief("Belief module")
```

**configs/top_decoder_test/top_decoder_test.py**：
```python
import m5
from m5.objects import *

root = Root(full_system=False)

# 先实例化所有子模块
root.ch_evidence = ChEvidence(default_bit_error_log_prob=13)
root.var_to_check = VarToCheck()
root.check_to_var = CheckToVar()
root.belief = Belief()

# 顶层模块传入子模块引用
root.top_decoder = TopDecoder(
    ch_evidence=root.ch_evidence,
    var_to_check=root.var_to_check,
    check_to_var=root.check_to_var,
    belief=root.belief,
    max_iterations=10
)

# TestGenerator通过事件调度测试
root.tester = TopDecoderTestGenerator(dut=root.top_decoder, clock_period=100)

m5.instantiate()
exit_event = m5.simulate()
```

# 3. 注意事项
## 3.1 子模块的process()方法必须为public

**重要**：如果顶层模块需要调用子模块的process()方法，该方法必须在子模块的头文件中声明为public，在使用子模块之前需要检查。

```cpp
// 错误：process()为private
class VarToCheck : public SimObject {
private:
    void process();  // 顶层模块无法调用
public:
    void setNeighbourChecks(uint64_t val);
    int32_t getVarToCheckMessage();
};

// 正确：process()为public
class VarToCheck : public SimObject {
public:
    void setNeighbourChecks(uint64_t val);
    void process();  // 顶层模块可以调用
    int32_t getVarToCheckMessage();
};
```


## 3.2 事件调度注意事项

**同一Event不能在同一时刻被schedule两次！**
```cpp
// 正确：在末尾统一调度
void
Module::processEvent()
{
    // 处理逻辑...
    if (!event.scheduled()) {
        schedule(event, curTick() + latency);
    }
}

// 错误：多个分支中调度
if (condition1) {
    schedule(event, ...);  // 可能已调度
} else {
    schedule(event, ...);  // 可能已调度
}
```

**多个模块同时调度可能冲突**：
- 如果TestGenerator和DUT都在tick=100调度，会冲突
- TestGenerator应先设置输入，DUT后处理
- 或者两者使用不同的时间间隔

## 3.3 等待周期与输出验证（重要！但应该先评估如果不需要则不必采用）

**问题**：时序逻辑模块的状态机转换需要多个时钟周期，测试时必须等待状态稳定后才能读取输出。

**解决方案**：在TestGenerator中添加wait_cycles机制

### 3.3.1 头文件添加成员变量
```cpp
class SpiTestGenerator : public SimObject {
private:
    void processEvent();
    EventFunctionWrapper event;
    const Tick clock_period;
    Spi& dut;
    size_t input_index;
    int wait_cycles;  // 等待周期计数器
    // ...
};
```

### 3.3.2 实现中等待并读取输出
```cpp
void
SpiTestGenerator::processEvent()
{
    if (wait_cycles > 0) {
        // 等待DUT处理
        wait_cycles--;
        // 读取并输出DUT状态
        DPRINTF(SpiTestGenerator, "OUTPUT: rx_data=0x%02x, rx_valid=%d, sck=%d, mosi=%d, cs=%d, busy=%d\n",
                dut.getRxData(), dut.getRxValid(), dut.getSck(), dut.getMosi(), dut.getCs(), dut.getBusy());
    } else {
        // 应用新的测试向量
        applyVector(input_index);
        input_index++;
        // 读取并输出DUT状态
        DPRINTF(SpiTestGenerator, "OUTPUT: rx_data=0x%02x, rx_valid=%d, sck=%d, mosi=%d, cs=%d, busy=%d\n",
                dut.getRxData(), dut.getRxValid(), dut.getSck(), dut.getMosi(), dut.getCs(), dut.getBusy());
    }

    if (!event.scheduled()) {
        schedule(event, curTick() + clock_period);
    }
}
```

### 3.3.3 applyVector中设置等待周期
```cpp
void
SpiTestGenerator::applyVector(size_t index)
{
    switch (index) {
        case 0:
            dut.setTxValid(1);
            // 简单状态跳转等待1-2周期
            wait_cycles = 2;
            break;
        case 3:
            // 完整传输需要等待8+周期
            wait_cycles = 10;
            break;
        // ...
    }
}
```
## 3.4 DPRINTF 输出格式规范

TestGenerator 的 DPRINTF 输出应统一遵循以下模式（以 riscv_core_test 为范例）：

**启动信息**：
```cpp
DPRINTF(XXXTestGenerator, "Starting xxx tests, total %d sequences\n", (int)testSequences.size());
```

**测试通过**（含描述）：
```cpp
DPRINTF(XXXTestGenerator, "Test %s PASSED %s\n", seq.id, seq.desc);
```

**测试失败**（含描述）：
```cpp
DPRINTF(XXXTestGenerator, "Test %s cycle %d FAILED  %s\n", seq.id, (int)currentCycle, seq.desc);
panic("XXXTestGenerator: Test %s cycle %d failed!", seq.id, (int)currentCycle);
```

**完成汇总**：
```cpp
DPRINTF(XXXTestGenerator, "Tests complete: %d passed, %d failed, %d total\n",
        (int)passed, (int)failed, (int)testSequences.size());
exitSimLoop("XXXTestGenerator complete");
```

## 3.5 格式说明符规范

**重要：`%zu` 不被 gem5 的 DPRINTF 支持！**

gem5 的 DPRINTF 内部使用自定义格式化，不支持 C99 的 `%zu`（size_t）。

**错误写法**（编译能通过，但运行时 `%zu` 会输出 `<bad format>`）：
```cpp
DPRINTF(XXXTestGenerator, "total %zu vectors\n", numVectors);  // ❌ 输出: total <bad format> vectors
```

**正确写法**（显式转型为 `int`，使用 `%d`）：
```cpp
DPRINTF(XXXTestGenerator, "total %d vectors\n", (int)numVectors);  // ✓ 输出: total 4 vectors
```

`size_t` 类型的变量（如 `current_test`、`numVectors`、`testVectors.size()` 等）在 DPRINTF 中一律：
1. 用 `(int)` 显式转型
2. 使用 `%d` 格式化

## 3.6 调试输出优化（重要！）

**问题**：DUT的`processEvent()`在每个tick都被调度执行，如果每次都打印调试信息，会产生大量无效输出。

**解决方案**：只在关键节点打印调试信息

**错误示例**（会产生大量无用输出）：
```cpp
void
Module::processEvent() {
    DPRINTF(Module, "Processing event, current state: %d\n", (int)currentState);  // 每tick都打印！
    // ... 处理逻辑
    if (!event.scheduled()) {
        schedule(event, curTick() + 1);
    }
}
```

**正确示例**（只在关键节点打印）：
```cpp
void
Module::processEvent() {
    switch (currentState) {
        case State::IDLE:
            if (startFlag) {
                DPRINTF(Module, "IDLE: start detected, transitioning to SETUP\n");
                startFlag = false;
                transitionState(State::SETUP);
            }
            break;
        case State::SETUP:
            DPRINTF(Module, "SETUP: complete, transitioning to TX\n");
            transitionState(State::TX);
            break;
        case State::TX:
            sendBit();
            bitCounter++;
            DPRINTF(Module, "TX: sent bit %d/8\n", bitCounter);
            if (bitCounter >= 8) {
                transitionState(State::RX);
            }
            break;
        // ...
    }

    if (!event.scheduled()) {
        schedule(event, curTick() + 1);
    }
}
