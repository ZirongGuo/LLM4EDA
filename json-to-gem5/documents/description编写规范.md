# Description 文件编写规范

本文档描述如何编写模块的 `decription.md` 功能描述文件。

## 文件命名

- 文件名：`decription.md`（固定）
- 编码：UTF-8
- 位置：与模块的 `.cc`、`.hh`、`.py`、SConscript 文件位于同一目录

## 文件结构

每个 description 文件应包含以下章节（按顺序）：

```
1. 模块名称
2. 逻辑类型
3. 功能概述
4. 端口定义
   - 输入信号
   - 输出信号
5. 状态机（可选）
6. 功能说明
7. Verilog 关联（可选）
```

---

## 各章节详细规范

### 1. 模块名称

**格式：**
```markdown
## 模块名称
模块名 (英文缩写)
```

**规范：**
- 列出模块的英文名称和缩写（如有）
- 缩写使用英文括号包裹

**示例：**
```markdown
## 模块名称
clint (Core Local Interruptor)
```
---

### 2. 逻辑类型

说明是组合逻辑还是时序逻辑
以及说明是子模块还是顶层模块

**重要说明（子模块时序逻辑）：**
- 子模块时序逻辑的 `process()` 函数调用本身就代表一个时钟上升沿
- **不需要**在子模块中实现时钟边沿检测（如 `prev_clk`、`clk_rising` 等）
- **不需要** `clk` 输入端口，顶层模块通过每个周期调用 `process()` 来驱动子模块
- 写操作直接在 `process()` 中执行，不需要判断时钟边沿

**示例：**
```markdown
## 逻辑类型
时序逻辑（子模块）

注意：作为子模块，其 process() 调用由顶层模块每个周期触发，代表一个时钟上升沿。
不需要 clk 输入端口和时钟边沿检测逻辑。
```

---

### 3. 功能概述

**格式：**
```markdown
## 功能概述
[功能描述文字]
```

**规范：**
- 1-3 句话概括模块功能
- 包含模块所属架构（如 RISC-V、ARM 等）
- 说明模块的主要职责
- 描述模块与其他组件的关系（上下游）

**示例：**
```markdown
## 功能概述
CLINT 模块是 RISC-V 架构中的核心本地中断控制器，负责处理同步中断（ECALL、EBREAK）、异步中断（外部中断）和异常返回（MRET）。它管理与 CSR（Control and Status Register）寄存器的交互，产生中断信号并控制处理器流水线。
```

### 4. 端口定义

#### 4.1 输入信号

**格式：**
```markdown
### 输入信号
| 端口名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| clk | 1 | input | 时钟信号 |
```

**规范：**
- 使用 Markdown 表格
- 列顺序：端口名 | 位宽 | 方向 | 描述
- 位宽格式：
  - 单比特：`1`
  - 向量：`N:M`（如 `31:0`）
  - 特殊格式需注明（如 `8` 表示 8 位）
- 方向：统一使用 `input` / `output`
- 描述：简洁明确，说明信号用途

**常用位宽格式：**
| 格式 | 含义 | 示例 |
|------|------|------|
| `1` | 单比特 | `1` |
| `N:M` | 向量位宽 | `31:0` 表示 32 位 |
| `N` | 标量位宽 | `8` 表示 8 位 |

**常见信号类型：**
| 信号类型 | 命名建议 | 描述要点 |
|----------|----------|----------|
| 时钟 | `clk` | 时钟信号 |
| 复位 | `rst` | 复位信号，注明有效电平 |
| 数据 | `data_i/o` | 数据信号，注明位宽 |
| 地址 | `addr_i/o` | 地址信号，注明位宽 |
| 使能 | `en_i/o` | 使能信号 |
| 标志 | `flag_i/o` | 标志信号 |
| 有效 | `valid_i/o` | 有效信号 |
| 就绪 | `ready_i/o` | 就绪信号 |
| 握手 | `req_i/o`, `ack_i/o` | 请求/应答信号 |

#### 4.2 输出信号

**格式：**
```markdown
### 输出信号
| 端口名 | 位宽 | 方向 | 描述 |
|--------|------|------|------|
| hold_flag_o | 1 | output | 流水线暂停标志 |
```

**规范：**
- 同输入信号规范
- 方向统一使用 `output`

### 5. 状态机

**格式：**
```markdown
## 状态机

### 状态机名称 (状态位宽)
- 状态名 (状态值): 状态描述
```

**规范：**
- 如果模块有多个状态机，按功能分组
- 每个状态列出名称、值、含义
- 状态值使用 Verilog 格式（如 `4'b0001`）
- 复杂状态机可附加状态转移说明

**示例：**
```markdown
## 状态机

### 中断状态机 (4 位)
- S_INT_IDLE (4'b0001): 空闲状态
- S_INT_SYNC_ASSERT (4'b0010): 同步中断响应
- S_INT_ASYNC_ASSERT (4'b0100): 异步中断响应
- S_INT_MRET (4'b1000): 异常返回
```

---

### 6. 功能说明

**格式：**
```markdown
## 功能说明

### 子功能标题
- 功能点1
- 功能点2
```

**规范：**
- 使用多级标题组织功能模块
- 每个子功能用列表项描述
- 描述具体的处理逻辑和行为
- 包含关键算法、协议、流程

**子功能分类建议：**

| 分类 | 描述内容 |
|------|----------|
| 初始化 | 上电复位、配置加载 |
| 主功能 | 核心业务逻辑 |
| 异常处理 | 错误检测、异常上报 |
| 状态管理 | 状态转换、条件触发 |
| 接口交互 | 握手协议、数据传输 |
| 功耗控制 | 时钟门控、电源管理 |

**示例：**
```markdown
## 功能说明

### 1. 同步中断处理
- 检测 ECALL、EBREAK 指令
- 等待除法操作完成后响应
- 保存当前指令地址到 MEPC
- 设置中断原因到 MCAUSE

### 2. 异步中断处理
- 检测外部中断标志
- 在全局中断使能时响应
- 处理除法正在进行的情况
```

---

### 7. Verilog 关联

**格式：**
```markdown
## Verilog 关联
- 关联的 Verilog 文件
- 宏定义依赖
- 共享信号说明
```

**规范：**
- 说明与原始 Verilog 文件的对应关系
- 列出依赖的宏定义文件
- 注明共享的内部信号或接口

**示例：**
```markdown
## Verilog 关联
- 对应原始模块：`src/verilog/clint.v`
- 包含 `defines.v` 中的宏定义
- 与 `csr_reg` 模块共享 CSR 接口
```

## 格式检查清单

编写完成后，请检查：

- [ ] 文件名为 `decription.md`
- [ ] 位于正确目录（与代码文件同级）
- [ ] 编码为 UTF-8
- [ ] 使用中文标点符号
- [ ] 表格格式正确
- [ ] 标题层级清晰
- [ ] 无错别字
- [ ] 信号位宽格式统一
- [ ] 方向统一使用 input/output
